The default snapshot repository stores all snapshots generated by data stream lifecycle management. The first repository you register in {{kib}} becomes the default automatically, and you can change which repository is the default at any time.

The default repository is marked with a **Default** badge in the repositories list and the repository details flyout.

To change the default repository from the **Repositories** list in {{kib}}:

1. Open the actions menu for the repository you want to use as the default.
2. Select **Set as default**.
3. In the **Change default repository?** dialog, select **Change default** to confirm.

You can also change the default when you edit a repository: turn on the **Set as default repository** switch, then save. If a different repository is already the default, a confirmation dialog appears before the change is applied.

The following constraints apply:

* The default repository cannot be removed. To remove it, first set another repository as the default.
* Read-only repositories cannot be set as the default.
* URL repositories cannot be set as the default.

To change the default repository from {{kib}}, you need the `cluster:admin/settings/update` [cluster privilege](elasticsearch://reference/elasticsearch/security-privileges.md#privileges-list-cluster). Without it, the **Default** badge is still visible, but the controls for changing the default are hidden. The default repository is stored in the `repositories.default_repository` cluster setting.
