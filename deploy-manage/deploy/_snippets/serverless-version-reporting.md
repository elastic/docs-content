In Serverless, the `GET /` root API always reports the next upcoming minor Elasticsearch release version at the time of the request, not the version currently deployed. Serverless is continuously updated and does not follow traditional release versioning. The `version.number` field exists to maintain compatibility with clients that require a semver value. Use `build_flavor: serverless` to confirm you're targeting a Serverless project rather than relying on the version number.
